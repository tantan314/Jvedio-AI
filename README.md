# Jvedio-AI
一个本地化的（JAV）电影管理/数据库系统

<img width="2591" height="1680" alt="image" src="https://github.com/user-attachments/assets/a43c4d65-b61f-4b6c-b52b-75e550992dc2" />


Jvedio AI是基于原Jvedio开源（C#/.NET）项目修改而来且持续维护的新一代JAV/电影管理系统。本项目已获得原项目作者授权，且已经剔除使用GPL 3.0等许可证的第三方库，从而保证其满足MIT许可协议。

原项目地址：https://github.com/hitchao/Jvedio

本软件除大量重构和优化了原版软件的代码及功能外，还额外新增了众多实用功能：

1. 基于本地AI模型的影片数据语义分析功能。
2. 基于本地AI模型的使用者用户画像功能。
3. 基于本地AI模型的个性化推荐功能。
4. 基于本地AI模型的同类影片推荐功能。
5. 基于本地AI模型的语义搜索功能。
6. 基于Serverless架构的云端字幕匹配和下载功能。

本项目旨在打造一款主要功能纯本地化、注重隐私、符合当下用户使用场景需求和习惯的影片管理系统。

注：本项目与原项目不同，不再提供对32位系统和纯32位CPU的支持。



除继承了Jvedio的整体框架、界面设计外，本项目通过对原项目的代码重构，对内存占用、稳定性、WPF UI虚拟化、磁盘I/O异步线程、UI响应、数据库写入乐观锁、数据读写防抖、AI模型向量化性能等方面进行了深度优化，实现了功能和性能开销上的平衡。

项目本身并不提供任何资源或资源获取渠道。所有数据均来源于用户本地存储资源，以及互联网公开数据源。

本项目基础功能可流畅运行在入门至主流配置的PC上（开发测试环境包含了Z87+4770K+16GB内存平台主机，以及9950X3D+X870E+48GB内存平台主机）。

AI分析、推荐、语义检索等功能，建议使用2020年后上市的6核+16GB以上内存的PC运行，以获得良好用户体验。

本项目中所有涉及AI模型数据分析的环节，均使用CPU运算，在用户本地环境中执行，不上传任何数据到任何远程服务器。

本项目中所有涉及网络连接的功能，均采用“远程数据+本地对比/分析”方式，不上传任何数据到任何远程服务器。

本项目并未维护原Jvedio项目中包含的影片元数据刮削功能。建议用户使用第三方刮削软件或插件，对本地已有影片进行刮削后，再将数据导入本项目/软件。

本项目当前的核心功能，是基于BGE-M3-dense-only模型的“单模型、单向量”架构实现的影片语义（元数据）分析、用户画像（随用户在软件内的操作行为实时更新）、影片推荐（影片 vs. 用户画像）、自然语义检索（类似RAG系统，但不含Generation环节，只展示检索结果）以及类似影片推荐功能。

该架构最大意义在于：
1. 证明Embedding模型即便缺乏特殊的语义训练材料，也能依靠正常语料训练库中的相关/类似语料，实现在AI模型针对跨/多语言语义内容向量在高维度空间内的自然聚拢。
2. 证明语义检索及召回（Semantic query & recall）并不需要复杂的架构、向量数据库以及多向量设计，仅需针对输入token进行适当清理和NLP格式化，以及恰当的MMR重排，也能媲美甚至超越“调包侠”拼凑出的RAG系统的检索精度。
3. 证明使用现代主流平台，不依赖GPU算力，依然能做到性能/响应可用乃至优秀的AI应用的工程落地。

若对本项目AI功能相关的核心算法及原理感兴趣，可阅读仓库中公开的“算法.html”文件。也欢迎有能力的用户为本项目提出各项优化或新功能建议。

致谢：本项目基于原Jvedio项目进行二次开发。若没有原项目优秀的数据库表和字段设计、功能规划，本项目也无法实现上述功能。同时感谢开源社区提供的各类第三方库，使得AI模型调用、浮点运算加速等工程能够相对简洁且顺利实现。

免责声明：本项目部分功能/UI设计针对特定类型影片进行优化，但项目/软件的各项功能并不只针对特定影片生效，而是广泛适用于任何附带.nfo元数据的视频文件管理、检索和推荐。项目/软件中的任何敏感数据均来源于用户本地存储数据以及互联网数据源，项目/软件不提供任何对敏感数据的分享功能。
