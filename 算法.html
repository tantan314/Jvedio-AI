Jvedio AI 推荐系统：算法与架构白皮书

版本： 4.0 (BGE-M3 Semantic Search & Metadata Optimized)
定位： 本地化、离线化工业级影视内容语义搜索引擎与推荐引擎
受众： 资深后端开发、算法工程师、推荐系统架构师

1. 系统架构概述 (System Architecture Overview)

本系统是一个完全离线化的语义搜索与推荐引擎。系统抛弃了传统的纯倒排索引与关键字精准/模糊匹配，采用多模态特征融合（NLP 语义 + 连续性数值参数）的高维混合张量表示法。

核心推荐管线（Pipeline）包含三大物理层：

  1. 特征工程与向量化层 (Embedding Layer)： 将非结构化文本与结构化数值映射为统一的 1106 维高维连续实数空间，深度适配 BGE-M3 语义模型。
  2. 画像对冲与生成层 (Profiling Layer)： 基于历史行为数据，利用对比学习与 TF-IDF 差分思想，分离用户的“隐性底色”与“显性偏好/厌恶”，生成 Master (正向) 与 Penalty (负向) 双向量作为用户画像。
  3. 召回与硬件加速层 (Retrieval & Hardware Acceleration)： 基于 System.Numerics.Vectors 的 AVX2 指令集+全内存SIMD浮点运算加速，融合数据稀疏性惩罚（Sparsity Penalty）与 MMR 重排，实现毫秒级精准响应。

2. 特征工程与向量化 (Feature Engineering & Vectorization)

本模块的目标是将单部影片的所有异构数据合并为一个定长的高维张量（$D = 1106$）。

2.1 文本语义轨道 (NLP Track)

  • 模型选择： 采用 BAAI/bge-m3 模型，原生支持多种长度文本的高效对称语义编码。
  • 对称编码与特征重排 (Symmetric Encoding & Metadata Reordering)：
    • 零前缀机制： BGE-M3 模型原生支持对称与非对称检索，不再需要人为的 passage: 和 query: 前缀，直接送入原始文本。
    • 拼接顺序优化： Plot (剧情) -> Genres (类型) -> Label (标签) -> Actor (演员) -> Series (系列) -> Title (标题)。将语义密度最高的剧情置于最前，标题置于最后，以最大化利用 Transformer 的自注意力机制（Self-Attention）与位置编码，避免信息截断。
    • 元数据精确清洗 (Metadata Cleaning)： 在进入大模型向量化前，引入独立的 blacklist_plot.txt以及blacklist_genre.txt 实现 100% 全字匹配拦截，彻底刮削数据中混入的广告词、水印或固定格式冗余词汇，确保输入给 Embedding 模型的文本语义绝对纯净。
    • 静态兜底剧情 (Dummy Plot)： 若原文本缺失剧情，系统自动使用标签和类型生成静态伪剧情，利用基础语法结构稀释标签浓度，彻底阻断自注意力机制对短文本标题的异常短路匹配，消除“空壳锚点漂移效应”。
  • 算法流程：
    1. 分词 (Tokenization)： 将文本切分为 Token，最大截断长度提升至 $L = 768$。
    2. 前向传播 (Forward Pass)： 提取模型最后一层隐藏状态，或直接读取 Vespa 优化后的二维池化输出，维度为 $[1, 1024]$。
    3. 池化 (Mean/CLS Pooling)： 结合 Attention Mask，输出稠密向量，将包含演员NLP信息的向量降维至 $[1024]$。
    4. 归一化： 执行严格的全局 $L2$ 归一化 $\frac{v}{\|v\|_2}$。

2.2 数值连续特征轨道 (Numerical Feature Track)

解决传统 NLP 模型对具体数字不敏感的物理缺陷。该轨道将少量的连续数值型特征，通过核方法升维至 82 维的稠密空间。

  • 输入数据： 包含 6 项演员原生参数与 3 项派生参数。
  • 核心算法：分位数锚点高斯激活 (Quantile-based Gaussian Kernel)
    • 对每一种数值特征，提取 $K$ 个分位数（如 10%, ..., 90%）作为高斯函数的均值 $\mu_i$（锚点），9个特征的锚点总数相加即为 82 维。
    • 自适应计算方差 $\sigma_i$（相邻锚点间距的平均值）。
    • 计算观测值 $x$ 在每个锚点上的激活强度：
      $$A_i = \exp\left(-\frac{(x - \mu_i)^2}{2\sigma_i^2}\right)$$
  • 聚合策略： 针对多演员情况执行高斯激活均值池化，最终进行局部 $L2$ 归一化。

2.3 混合张量融合与缩放 (Hybrid Fusion & Scaling)

将上述两个轨道的向量拼接到同一向量空间（$1024 + 82 = 1106$ 维）。为防止高维空间塌缩与特征淹没，采用特定权重进行 Scale。

  • 降维合并： 采用 1106 维替代了旧有的臃肿架构，保证语义富集度的同时，依托 Buffer.BlockCopy 序列化压缩 SQLite 磁盘体积。

3. 召回与语义搜索机制 (Semantic Search & Retrieval)

3.1 AVX2/SIMD 硬件加速 (Hardware Acceleration)

为了在本地客户端实现“全内存无倒排穷举比对”，抛弃了低效的标量计算 for 循环，底层直接使用 System.Numerics.Vectors<float> 进行汇编级硬件加速。

  • 自适应降级： JIT 编译器动态评估 CPU 能力，支持 AVX2 的 CPU 单周期可吞吐 8 个 Float 并行乘加（256-bit）；若硬件不支持，可无缝平滑降级至 SSE2 甚至标量运算，拒绝异常崩溃。

3.2 元数据稀疏性惩罚 (Metadata Sparsity Penalty)

工程缺陷修复： 解决 Embedding 模型中“空壳锚点漂移效应”。当某部影片完全缺失 Title、Plot、Genre 等字段时，其生成的向量会退化为空洞特征，与任意输入 Query 极易产生畸形的高余弦相似度（假阳性匹配）。

  • 在召回阶段通过 SQL 实时统计元数据长度：LENGTH(Title) + LENGTH(Plot) + LENGTH(Genre)
  • 若总长度 $\le 5$：进行 极度降权 (Score $\times 0.1$)，强制剔除出 Top-N。
  • 若总长度 $\le 15$：进行 温和降权 (Score $\times 0.6$)。

3.3 独立相似影片查询轨道 (Isolated Similar Movies Track)

业务痛点与取舍： 传统的基于混合张量（1024 维文本 + 82 维连续特征）的 KNN 最近邻搜索，在处理“查找相似影片”任务时存在固有缺陷：由于演员身材特征（数值轨道）的权重干扰，两部剧情/题材完全不相干的影片，可能仅仅因为主演的身材数值高度重合，而产生畸形的高相似度，导致“特征串味”。

  • 动态降维查库策略 (Dynamic Dimensionality Reduction)： 为解决此问题，系统在执行“查找相似影片”与“语义搜索”时，实施了不对称查询策略。
  • 纯文本语义重构： 系统不使用数据库中该影片现成的 1106 维向量，而是基于该影片的纯文本特征（剥离所有身体数值），实时调用大模型生成一个 1024 维的 纯文本临时查询向量。
  • 自动对齐降维比对： 凭借点积运算与余弦相似度的维度自适应特性，1024 维的查询向量在与全库 1106 维的向量池比对时，会自动忽略全库向量的后 82 维数值分量。这从根本的数学物理层面上，彻底排除了身材数据对“相似度查询”的干扰，让相似影片完全回归其原始的题材与剧情语义关联。

4. 智能推荐引擎与多样性重排 (AI Recommendation & Diversity Engine)

推荐引擎不采用全库暴力穷举，而是通过严密的统计学公式与概率抽样，在保证“极高准确率”的前提下，最大化“盲盒发现感（Serendipity）”。

4.1 平方根衰减真随机抽样 (Square Root Decay True Random Sampling)

为了解决当影库规模极大时，固定比例抽样会导致头部神作被过度曝光的“信息茧房”问题，系统引入了随影库规模动态调整的平方根衰减法：

  • 单批次采样公式：
    $$SampleSize = \min\left(400, \max\left(4 \times BatchSelect, \sqrt{N} \times 2.5\right)\right)$$
    其中 $N$ 为全库已向量化影片总数。
  • 物理意义： 当 $N=300$ 时，单次抽样率为 14.3%；当 $N=16000$ 时，单次抽样率断崖式跌落至 1.9%。这完美剥夺了大库中那几十部 5 星神作“靠海量遍历次次被抽中”的特权，让算法每次只能在冰山一角中做局部寻优。
  • UUID 级真随机洗牌： 弃用标准 Random.Next() 伪随机发生器，采用 Guid.NewGuid() 进行 $O(N)$ 复杂度的系统底层真随机打乱，确保每一部影片进入采样池的数学概率绝对均等。

4.2 绝对值加权复核打分 (Additive Weighted Scoring, AWS)

抛弃了粗糙的排序融合（RRF），采用极其严密的浮点数加权体系，彻底消灭“同星级分数平局”现象，建立绝对的阶级护城河：

  • 基础相似度 (Similarity)： $Sim = \text{CosineSimilarity}(\text{MasterVector}, \text{FilmVector})$，并受到负向画像 `PenaltyVector` 的特征惩罚。
  • 独立质量分 (Quality Score)： $Q = (\frac{Grade}{5} \times 0.7 + \frac{Rating}{100} \times 0.3) \times \text{ViewPenalty}$ （基于观看次数的疲劳衰减）。
  • 最终复核得分 (PreScore)：
    $$PreScore = 0.85 \times Sim + 0.15 \times Q$$
    此公式确立了“语义相似度绝对主导，高星级评分锦上添花”的底层基调。

4.3 动态对数扩池与 MMR 重排 (Logarithmic Expansion & Maximal Marginal Relevance)

为打破那少数几部“T0级神作（极高相似度+5星）”对推荐结果首屏的绝对霸榜，系统重新激活了 MMR 算法，并结合了自适应的对数级蓄水池扩容机制：

  1. 对数级平滑扩容 (Logarithmic Pool Expansion)： 在进入最终挑选前，系统不直接抓取 50 部，而是根据库大小计算一个动态乘数：
    $$Multiplier = \max(2.0, \min(5.0, \log_{10}(N)))$$
    对于 16000 部的大库，乘数为 4.2，系统会先挑选出 $50 \times 4.2 = 210$ 部高质量候选影片。这为底部的打散去重算法提供了极其充足的“弹药库”。
  2. 演员配额拦截 (Actor Quota)： 硬性规定单一演员在最终结果中的出场率不得超过 $\max(1, TopN \times 10\%)$，强制截断屠版现象。
  3. MMR 多样性惩罚打散： 算法从 210 部候选大名单中，逐一挑选出 50 部。每挑选一部，都会对其余影片进行惩罚：
    $$MMR\_Score = \lambda \cdot PreScore - (1-\lambda) \cdot \max_{s \in Selected} Cosine(Film, s)$$
    (当前系统设定 $\lambda = 0.7$)。这意味着如果某部神作与已经选出的神作在特征上高度同质化（例如题材完全一致），它将被重重扣分并被无情踢出名单。从根本的几何空间上，确保了展现结果既有极高水准，又绝不雷同。
